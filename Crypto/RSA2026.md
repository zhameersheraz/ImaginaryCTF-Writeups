## Background Knowledge (Read This First!)

### What is RSA?

**RSA** encrypts a message `m` as:

```
c = m^e mod n
```

To decrypt, you normally need the private key `d`. But the exponent `e` in this challenge has a special hidden structure that breaks everything.

### The Vulnerability — Exponent of the form `k*(p-1) + 1`

If the exponent `e` happens to be of the form:

```
e = k * (p-1) + 1   for some small prime p where p divides n
```

Then by **Fermat's Little Theorem**, for that prime `p`:

```
m^e mod n mod p  =  m^1 mod p  =  m mod p
```

This means **computing `c mod p` directly gives you `m mod p`** — no private key needed!

### Why does this help?

Do this for many small primes `p_1, p_2, p_3, ...` and collect:

```
m mod p_1
m mod p_2
m mod p_3
```

Then the **Chinese Remainder Theorem (CRT)** stitches them all back together to reconstruct the full message `m` (the flag).

## Key Takeaways

- RSA exponents of the form `k*(p-1)+1` leak `m mod p` for free via Fermat's Little Theorem
- Collecting enough small-prime residues and applying **CRT** fully reconstructs the plaintext
- Short flags are dangerous in poorly designed RSA — always pad messages properly
- Always factor the exponent when you see an unusual RSA challenge
